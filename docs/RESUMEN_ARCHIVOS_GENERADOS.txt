================================================================================
                       CÓMO GENERAR EL DIAGRAMA
================================================================================

OPCIÓN 1: Instalar Graphviz y generar (RECOMENDADO)
------------------------------------------------------
sudo dnf install graphviz
./generate_diagram.sh

Resultado: md_document_editor_v2_class_diagram.png


OPCIÓN 2: VSCode con extensión PlantUML
-----------------------------------------
1. Instalar extensión "PlantUML" en VSCode
2. Abrir md_document_editor_v2_class_diagram.puml
3. Presionar Alt+D para preview
4. Click derecho → "Export Current Diagram"


OPCIÓN 3: Online (sin instalación)
------------------------------------
1. Ir a: https://www.plantuml.com/plantuml/uml/
2. Copiar contenido de md_document_editor_v2_class_diagram.puml
3. Pegar en el editor web
4. Descargar PNG/SVG

================================================================================
                           ESTRUCTURA DEL PROYECTO
================================================================================

CAPAS DE LA ARQUITECTURA:

1. 🟠 Vista (UI Layer)
   - MDDocumentEditor (RecycleView)
   - SelectableRecycleBoxLayout
   - MDDocumentLineEditor

2. 🔵 Gestión de Estado (State Layer)
   - DocumentStateManager (StateManager Pattern)
   - LineState (Immutable)
   - StateChangeEvent

3. 🟢 Servicios (Service Layer)
   - LineService (operaciones de líneas)
   - SelectionService (selección)
   - NavigationService (navegación)
   - FilterService (filtrado con filter_up)

4. 🟡 Modelo de Documento (Model Layer)
   - MDDocument
   - MDLine (Linked List)
   - MD_LINE_TYPE (Enum)

5. 🔴 Utilidades
   - UndoManager
   - TranslateMarkdownToKVMarkup

================================================================================
                      CARACTERÍSTICAS PRINCIPALES
================================================================================

✅ Estado Centralizado (FASE 1)
   - Single Source of Truth
   - Estados inmutables
   - Observer Pattern
   - Validación automática de invariantes

✅ Service Layer (FASE 2)
   - Lógica de negocio encapsulada
   - Servicios especializados
   - Fácil de testear
   - Código limpio

✅ FilterService Completo (FASE 3)
   - Filtrado por texto
   - Inclusión de títulos padre (filter_up)
   - Filtros personalizados
   - Gestión de visibilidad
